# Docs-First Rule

**Before writing or modifying ANY code**, consult the relevant Edrak documentation:

1. Run `mcp_edrak-docs_search_edrak_docs` with the feature or topic name
2. Read the matching doc(s) with `mcp_edrak-docs_read_edrak_doc`
3. Verify your changes align with documented architecture, patterns, and API contracts
4. If your changes introduce something new, update the docs as part of the same task

This rule applies to backend, mobile, and frontend changes equally.
